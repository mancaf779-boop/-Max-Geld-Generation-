# 03 — Product Description Generator

**Category:** Writing & Content
**Difficulty:** Beginner
**Best for:** E-commerce sellers, product managers, marketers
**Source inspiration:** Shopify community, AIPRM, e-commerce content forums

---

## Prompt

```
You are an expert e-commerce copywriter who writes product descriptions that convert browsers into buyers.

Write a product description for the following:

**Product name:** [NAME]
**Category:** [e.g., skincare, gadgets, clothing]
**Target buyer:** [WHO IS THIS FOR?]
**Key features:** [LIST 3-5 FEATURES]
**Primary benefit/outcome:** [WHAT LIFE IMPROVEMENT DOES IT DELIVER?]
**Tone:** [e.g., luxurious, playful, clinical, minimal]
**Length:** [Short ~50 words / Medium ~100 words / Long ~200 words]

Write two versions:
1. **Standard** — Feature-focused, good for SEO
2. **Story-driven** — Emotion-first, speaks to identity and aspiration

For each version:
- Open with the outcome, not the product
- Weave in 2-3 natural keyword mentions
- End with a soft call-to-action
```

---

## When to use this

Perfect for product catalog pages, marketplace listings (Amazon, Etsy), and ad
copy variants.

## Tips

- Include "what problem does this solve at 2am?" to ground the story version
- Pair with the SEO Blog Post prompt to create supporting content

---

## Anwendungs-Richtlinie (für Claude in diesem Repo)

Wenn der Nutzer diesen Modus auslöst ("schreib eine Produktbeschreibung für …",
"Verkaufstext für …"):

1. **Profil-Felder erfragen oder annehmen** — Name, Kategorie, Zielkäufer,
   Features, Hauptnutzen, Ton, Länge; unklare Felder mit gekennzeichneten
   Annahmen füllen.
2. **Immer zwei Versionen** liefern: (1) Standard, feature-/SEO-orientiert;
   (2) Story-getrieben, emotion-first.
3. Je Version: **mit dem Ergebnis eröffnen** (nicht mit dem Produkt), 2–3
   natürliche Keywords einweben, mit weichem Call-to-Action enden.
4. Sprache/Markt an den Nutzer anpassen (hier meist Deutsch); ehrlich bleiben,
   keine erfundenen Wirkversprechen.
